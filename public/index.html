What makes this different

Most slide tooling assumes you bring a polished template. In practice, most people don't. They have a folder of decks that share a brand identity by author discipline, with no underlying master/layout structure to anchor a renderer to.

slide-publisher handles both cases. The plugin classifies the file you supply — is this a structured template, or a deck-with-implicit-pattern? — and routes accordingly:

Real template

Extract the layout catalog and style tokens directly. The extractor reads the layout names, master XML, and theme XML; the validator audits across six structural criteria.

Deck-with-implicit-pattern

Cluster the recurring visual patterns, synthesize a candidate template, then extract from that. The user reviews the synthesized template before locking it in.

Mixed

Surface both options to the user with the classifier's diagnostic signals. They choose.

Auto-remediation

When the validator flags issues, the auto-remediator applies deterministic fixes (layout renaming, theme alignment) and re-validates. Every change lands in an audit log.

The pipeline

Each step is a Cowork skill backed by a Python adapter. The skills compose; the adapters share a common module (_common.py) for IR catalogs, heuristics, and the loss-manifest dataclass.

The story IR

The compiler refuses to draft on a vague prompt. Audience, throughline, evidence anchor — supply those, and the IR is the durable artifact every renderer reads.

---
audience:
  primary: "VP of Design, Series C"
  prior_knowledge: warm
throughline: "Our growth model has plateaued; the next $40M is in the middle market."
arc: provocation
evidence_anchor: framework
duration_min: 20
---

## hook
The growth model that got us here will not get us to the next milestone.

## context
Enterprise pipeline has plateaued — four straight quarters flat at 8-10 deals.

## claim
Three reasons the middle market is the next motion: market shape, sales cycle, competitive window.

Install

cowork plugin install slide-publisher
cowork plugin grant slide-publisher --mcp figma
cowork run template-setup

The template-setup wizard is a one-time install workflow. It runs the classifier, extracts or synthesizes a template profile depending on what you supplied, validates the result against six structural criteria, and persists the profile at ~/.cowork/plugins/slide-publisher/profile.yaml.

Compile and render

cowork run story-compiler --input my-talk.outline.md --out my-talk.ir.yaml
cowork run slide-ir-validator --strict my-talk.ir.yaml
cowork run render-pptx --ir my-talk.ir.yaml --out my-talk.pptx
cowork run render-figma --ir my-talk.ir.yaml

Each renderer emits a loss manifest naming what was preserved exactly, what was preserved with degradation, what was dropped, and what was annotated.
